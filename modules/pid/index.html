01. UC-001¶

Use Case Identification and History

Use Case ID:

UC-001

Use Case Name:

Melakukan Login

Version No:

1

End Objective:

User berhasil Login

Created By:

BTI

On (date):

21 Mei 2021

Last Updated By:

User/Actor:

Dosen, Kaprodi, LPPI, GJM, Reviewer, Dekan, Bendahara Fakultas, Keuangan, WR1, WR2.

Business Owner:

UMS

Preconditions

Sistem menampilkan form login.

Basic Flow

Step

User Actions

System Actions

1

User login dengan menggunakan CAS menginputkan UniID/NIM dan password.

Sistem mengautentifikasi yang sudah diinputkan oleh user.

2

User mengklik button login.

Sistem menampilkan menu sesuai dengan akses level.

Alternate Flow

Step

User Actions

System Actions

1

User dapat Login menggunakan Gmail

Sistem menampilkan form Login menggunakan Gmail.

2

User memilih/memasukkan akun Gmail untuk Login.

Sistem memberikan konfirmasi keberhasilan login dengan sebuah message box, kemudian menampilkan halaman utama MyUMS yang berisi tombol untuk menuju ke menu lain.

Exception Flow

Step

User Actions

System Actions

1

UniUD/NIM dan password tidak valid.

Sistem memberikan peringatan berupa pop-up bahwa UniID/NIM dan password tidak sesuai.

Post conditions

Sistem memberikan konfirmasi keberhasilan login dengan sebuah message box, kemudian menampilkan halaman utama MyUMS yang berisi tombol untuk menuju ke menu lain.

Special Requirements

Business Rules

Untuk login hanya bisa diakses oleh user pada unit yang ada di UMS saja.

Other Notes (Assumptions, Issues,)

02. UC-002¶

Use Case Identification and History

Use Case ID:

UC-002

Use Case Name:

Mengajukan Usulan Hibah PID

Version No:

1

End Objective:

User berhasil mengajukan usulan hibah PID

Created By:

BTI

On (date):

21 Mei 2021

Last Updated By:

User/Actor:

Dosen, Kaprodi

Business Owner:

UMS

Preconditions

User harus login dan melakukan pengisian formulir pengajuan Hibah PID terlebih dahulu.

Basic Flow

Step

User Actions

System Actions

1

User mengklik button Ajukan Proposal Baru/Ajuan Baru.

Sistem menampilkan form pengajuan Hibah PID.

2

User mengisi formulir pengajuan hibah PID serta mengupload file yang dibutuhkan.

Sistem merekam data yang diinputkan.

3

User mengklik button Ajukan Proposal/Ajukan Seminar.

System menyimpan data ajuan Hibah PID dan user menunggu proses approval dari Admin Hibah.

Alternate Flow

Step

User Actions

System Actions

1

Exception Flow

Step

User Actions

System Actions

1

User tidak bisa mengajukan usulan Hibah PID jika sudah mencapai batas maksimal.

Sistem akan menampilkan pop-up yang memberitahu bahwa User sudah mencapai batas maksimum pengajuan.

2

User tidak mengisi satu atau lebih kolom dalam fromulir pengajuan Hibah PID.

Sistem menampilkan alert jika user belum mengisi kolom dalam formulir.

Post conditions

User berhasil mengajukan usulan Hibah PID

Special Requirements

Business Rules

Terdapat batas maksimum pengajuan usulan Hibah PID. Untuk Hibah PID Penelitian dan Pengabdian maksimal sekali dalam satu periode. Hibah PID Seminar (Non-Scopus dan Scopus) maksimal dananya dalam 1 periode adalah Rp6.000.000

Semua kolom dalam formulir pengajuan Hibah PID wajib diisi.

Other Notes (Assumptions, Issues,)

Terdapat 4 kategori atau sub-menu dari Hibah PID, yaitu: Penelitian, Pengabdian, Seminar Non-Scopus, dan Seminar Scopus.

User bisa memilih jenis ajuan Hibah PID yang dikehendaki.
