01. UC-001¶

Use Case Identification and History

Use Case ID:

UC-001

Use Case Name:

Melakukan Login

Version No:

1

End Objective:

User berhasil Login

Created By:

BTI

On (date):

22 Mei 2021

Last Updated By:

User/Actor:

Dosen / Kaprodi / Dekan / LPPI / WR I / WR II / Keuangan

Business Owner:

UMS

Preconditions

Sistem menampilkan form login.

Basic Flow

Step

User Actions

System Actions

1

User login menggunakan CAS dengan menginputkan UniID/NIM dan password.

Sistem mengautentifikasi yang sudah diinputkan oleh user.

2

User mengklik button Login.

Sistem menampilkan menu sesuai dengan akses level.

Alternate Flow

Step

User Actions

System Actions

1

User mengklik button masuk dengan google.

Sistem menampilkan form login menggunakan akun google.

2

User memilih akun google yang akan digunakan.

Sistem memberikan konfirmasi keberhasilan login dengan sebuah message box

kemudian menampilkan halaman utama MyUMS yang berisi tombol untuk menuju ke menu lain.

Exception Flow

Step

User Actions

System Actions

1

UniID/NIM dan password tidak valid.

Sistem memberikan peringatan berupa pop-up bahwa UniID/NIM dan password tidak sesuai.

Post conditions

Sistem memberikan konfirmasi keberhasilan login dengan sebuah message box kemudian menampilkan halaman utama MyUMS yang berisi tombol untuk menuju ke menu yang lain.

Special Requirements

Business Rules

Untuk login hanya bisa diakses oleh user pada unit yang ada di UMS saja.

Other Notes (Assumptions, Issues,)

02. UC-002¶

Use Case Identification and History

Use Case ID:

UC-002

Use Case Name:

Mengajukan Usulan Insentif

Version No:

1

End Objective:

Berhasil mengajukan usulan insentif

Created By:

BTI

On (date):

22 Mei 2021

Last Updated By:

User/Actor:

Dosen / Kaprodi

Business Owner:

UMS

Preconditions

User telah login dan berada disalah satu halaman sub menu insentif akademik.

Basic Flow

Step

User Actions

System Actions

1

User mengklik button ajukan usulan insentif baru.

Sistem menampilkan form pengajuan usulan insentif baru.

2

User menginputkan data sesuai dengan form dan mengupload file pengajuan yang dibutuhkan.

Sistem merekam data yang diinputkan user.

3

User mengklik button ajukan insentif.

Sistem menyimpan, menampilkan data usulan insentif baru, dan user menunggu approval dari Kaprodi, LPPI, WR I, dan WR II.

Alternate Flow

Step

User Actions

System Actions

1

Exception Flow

Exception Flow

Step

User Actions

System Actions

1

User tidak menginputkan satu atau lebih kolom dalam form pengajuan insentif akademik.

Sistem memberikan alert bahwa user belum menginputkan data pada form.

Post conditions

User berhasil mengajukan usulan insentif baru.

Special Requirements

Business Rules

Semua kolom pada form ajuan insentif akademik harus diisi semua.

User bisa mengajukan ajuan insentif sebanyak-banyaknya.

Other Notes (Assumptions, Issues,)

Terdapat 5 kategori pilihan ajuan insentif didalam sub-menu insentif akademik, yaitu jurnal, prosiding, koran, book, book chapter, dan keynote speaker (arsip).

User bisa memilih ajuan insentif yang dikehendaki.

Insentif keynote speaker (arsip) tidaak membutuhkan approval dari admin insentif.

Step

User Actions

System Actions

System Actions

2

User mengklik button tolak.

Step

User Actions

System Actions

1

Post conditions
